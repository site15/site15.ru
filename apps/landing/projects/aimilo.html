Aimilo

Платформа персонального AI-контента: персонализированные открытки, анимации и песни через обычный чат без регистрации.

TL;DR

Aimilo — это коммерческая AI-платформа для создания персонализированного медиаконтента.

Пользователь описывает повод, загружает фотографии и рассказывает про человека — ИИ создаёт уникальный подарок: изображение, анимацию, песню или анимацию с песней.

Всё работает через обычный чат без регистрации. После оплаты пользователь получает результат без водяных знаков.

Проект в production, есть реальные пользователи и онлайн-оплата. Это SaaS, созданный практически одним человеком.

Почему появился проект

Я более двадцати лет строил сложные системы для других. Работал в больших проектах, проектировал микросервисные архитектуры, внедрял Kubernetes. Но в какой-то момент захотелось создать собственный продукт — от идеи до production, полностью самому.

Идея Aimilo родилась из простого наблюдения: люди не хотят пользоваться AI напрямую. Они не хотят писать промпты, выбирать модели, настраивать параметры. Они хотят решить свою задачу — поздравить маму, сделать подарок жене, удивить друга.

Так появился Aimilo — сервис, где весь AI скрыт за простым интерфейсом чата. Пользователь просто говорит, что хочет, и система делает всё остальное.

Главная идея

Aimilo — это не сервис открыток. Открытки — лишь первый продукт.

Главная идея значительно шире: платформа персонального AI-контента.

Сегодня:

  • Анимации с фотографиями
  • Песни для конкретного человека
  • Анимации с песней

Завтра:

  • Семейные фильмы
  • Персональные истории
  • Детские книги
  • Цифровые воспоминания
  • Персональные мультфильмы
  • Виртуальные персонажи
  • AI-подарки любого формата

Мы продаём не картинки и не песни. Мы продаём эмоции. Люди покупают реакцию другого человека — и это ключевое отличие от всех AI-генераторов контента.

Почему существующие AI идут не туда

Большинство AI-продуктов сегодня сосредоточены на технологии. Они показывают, какие модели используют, какие параметры настраивают, какое качество генерации.

Но пользователю всё это не интересно. Человек не хочет думать о том, какая модель лучше — GPT-4 или Gemini. Он хочет получить результат, который вызовет эмоцию у конкретного человека.

Интернет сделал массовым одинаковый контент. AI делает возможным массовое создание уникального контента. Не миллионов одинаковых картинок — а миллионов совершенно разных подарков.

Aimilo построен на этом принципе: интерфейс важнее модели. Технология должна быть невидимой. Через несколько лет никто не будет говорить «я использовал GPT» — как сейчас никто не говорит «я использовал TCP/IP» или «я использовал JPEG». Технология исчезает. Остаётся результат.

Архитектура и стек

Aimilo построен как классический SaaS с микросервисной архитектурой:

Angular Frontend
    ↕ REST API
NestJS Backend
    ↕
PostgreSQL + NATS JetStream
    ↕
AI-конвейер (OpenAI, Gemini, Suno)

Backend — NestJS

Основной бэкенд на NestJS обрабатывает пользовательские запросы, управляет сессиями, платежами и очередями. Модульная архитектура позволяет независимо развивать каждый аспект системы.

Frontend — Angular

Фронтенд на Angular предоставляет интерфейс чата для взаимодействия с пользователем. Минималистичный дизайн, фокус на контенте, никаких лишних элементов.

Очереди — NATS JetStream

AI-генерация — процесс небыстрый. NATS JetStream используется для асинхронной обработки запросов: пользователь получает уведомление о готовности, а не ждёт у экрана.

AI-конвейер

Система интегрирует несколько AI-сервисов: OpenAI для генерации изображений и текста, Gemini для дополнительной обработки, Suno для создания песен. Каждый запрос проходит через цепочку промптов, прежде чем результат будет готов.

Как это работает

Пользовательский путь максимально прост:

  1. Открывает чат — без регистрации, без ввода email, без социальных сетей
  2. Описывает повод — день рождения, юбилей, просто тёплые слова
  3. Загружает фотографии — если хочет добавить портрет в дизайн
  4. Рассказывает про человека — его интересы, характер, что важно
  5. Выбирает формат — изображение, анимация, песня или анимация с песней
  6. Оплачивает — онлайн-оплата, без подписок
  7. Получает результат — уникальный контент без водяных знаков

Весь AI-конвейер работает за кулисами. Пользователь не видит промптов, моделей, параметров — только результат.

Ключевые технические решения

AI-конвейер обработки запросов

Каждый пользовательский запрос проходит через многоэтапный конвейер: анализ описания человека, генерация промптов для изображений, создание текста песни, компоновка анимации. Каждый этап — отдельный AI-вызов с оптимизированным промптом.

Система версионирования промптов

Одна из ключевых инженерных особенностей — система версионирования AI-промптов с возможностью отката. Каждый промпт версионируется, что позволяет отслеживать, какая версия дала лучший результат, и при необходимости откатываться к предыдущей.

Антифрод-система

Для защиты от злоупотреблений реализована антифрод-система на основе анализа fingerprint и IP. Это не сложная ML-система, а набор эвристик, которые эффективно отсекают большинство попыток обхода оплаты.

Платёжная система

Интеграция с платёжным провайдером для онлайн-оплаты. Без подписок — разовая оплата за конкретный результат. Это осознанное решение: люди не хотят ещё одну подписку, они хотят заплатить один раз и получить подарок.

Почему один человек

Весь проект создан практически одним человеком. Это не подвиг — это следствие глубокого понимания продукта.

Когда ты сам проектируешь архитектуру, сам пишешь backend и frontend, сам интегрируешь AI-сервисы — ты принимаешь решения быстрее. Не нужно обсуждать, не нужно согласовывать, не нужно ждать.

Конечно, ИИ-ассистенты значительно помогали с написанием кода. Но архитектурные решения, выбор стека, дизайн API, UX-решения — всё это результат двадцатилетнего опыта в разработке.

Что делает Aimilo особенным

  • Без регистрации — пользователь может сразу начать создавать подарок. Никаких форм, никаких подтверждений email.
  • Через обычный чат — интерфейс, который все знают. Не нужно учиться пользоваться.
  • Персонализация — каждый результат создаётся для конкретного человека. Не шаблон, а уникальный контент.
  • Несколько форматов — изображения, анимации, песни. Пользователь выбирает, что лучше подходит.
  • Без водяных знаков — после оплаты пользователь получает чистый результат, который можно отправить в любом мессенджере.
  • Реальный production — это не демо и не пет-проект. Это работающий сервис с реальными пользователями и оплатой.

Философия проекта

Почему люди вообще дарят подарки? Потому что хотят вызвать эмоцию. Подарок ценен не сам по себе — ценна реакция человека, который его получает.

Бумажные открытки исчезают не потому, что они плохие — а потому что цифровой мир предлагает новые способы выразить чувства. Цифровые подарки становятся лучше физических не потому что они «цифровые», а потому что они могут быть персонализированы так, как это невозможно в физическом мире.

AI позволяет персонализировать эмоции. Не миллионов одинаковых картинок — а миллионов совершенно разных подарков. Каждый — для конкретного человека, с его фотографиями, его интересами, его историей.

Текущий статус

Это SaaS — разовая оплата за результат. Без подписок, без скрытых платежей.

Проект активно развивается. Добавляются новые форматы контента, улучшается качество генерации, оптимизируется AI-конвейер.
