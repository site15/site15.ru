OpWork

Платформа для поиска работы с открытым исходным кодом на NestJS + Vue 3 + Prisma + PostgreSQL.

TL;DR

За примерно 4 месяца частичной занятости я создал платформу для поиска работы с открытым исходным кодом — OpWork.

Платформа предоставляет полный цикл взаимодействия: соискатели создают резюме и откликаются на вакансии, работодатели публикуют вакансии и управляют откликами.

Код полностью открыт — сделал это, чтобы другим разработчикам могло пригодиться. Планировал на базе этой базы сделать проект для поиска работы и специалистов именно для NestJS-разработчиков, но сейчас проект поставлен на паузу — нет времени.

С чего всё началось

Идея родилась из желания создать open-source альтернативу существующим платформам по поиску работы. Большинство рекрутинговых сервисов — закрытые системы с непрозрачными алгоритмами ранжирования.

Мне хотелось сделать платформу, где весь код открыт, где можно понять, как работает поиск, как устроена модель данных, как организованы бизнес-процессы. Просто открыть код — может кому-то пригодится как база или референс.

Проект также стал полигоном для отработки архитектурных решений на NestJS + Vue 3 + Prisma — стеке, который я активно использую в работе.

Архитектура и ключевая идея

OpWork построен по классической трёхуровневой архитектуре:

Vue 3 Frontend (Ant Design Vue)
    ↕ REST API
NestJS Backend (Fastify)
    ↕ Prisma ORM
PostgreSQL

Ключевая архитектурная особенность — двойная роль пользователей. При регистрации создаются два профиля: соискатель и компания. Пользователь может переключаться между ними, что упрощает взаимодействие.

Все DTO генерируются автоматически из Prisma-схемы с помощью кастомного генератора prisma-generator-nestjs-dto, что обеспечивает строгую типизацию на всём пути от базы данных до фронтенда.

Почему стек именно такой

Backend — NestJS + Fastify

NestJS — мой основной фреймворк для бэкенда. Fastify выбран как высокопроизводительная альтернатива Express. Модульная архитектура NestJS идеально подходит для проектов с чётким разделением доменов.

Frontend — Vue 3 + Ant Design Vue

Для фронтенда выбран Vue Vben Admin — зрелый шаблон на Vue 3 с Ant Design Vue. Это позволило быстро собрать полноценный UI с таблицами, фильтрами, формами и маршрутизацией без написания всего с нуля.

ORM — Prisma

Prisma обеспечивает типобезопасную работу с базой данных, удобные миграции и автоматическую генерацию клиента. Схема в ~1900 строк описывает всю модель данных платформы.

Основные функциональные блоки

Для соискателей

  • Создание и управление резюме с образованием, опытом и навыками
  • Поиск вакансий с фильтрацией по навыкам, типу занятости, формату работы
  • Отклики на вакансии с сопроводительным письмом
  • Отслеживание статуса заявок
  • Система уведомлений об изменениях

Для работодателей

  • Управление профилем компании
  • Создание и публикация вакансий с тегами и навыками
  • Просмотр и управление заявками соискателей
  • Фильтрация заявок по статусам
  • Добавление примечаний к заявкам

Модель данных

Prisma-схема содержит ~1900 строк и описывает полноценную модель рекрутинговой платформы:

enum OpWorkUserType {
  JOB_SEEKER   // Соискатель
  EMPLOYER     // Работодатель
  ADMIN        // Администратор
}

enum OpWorkJobStatus {
  DRAFT → ACTIVE → PAUSED → CLOSED → ARCHIVED
}

enum OpWorkEmploymentType {
  FULL_TIME | PART_TIME | CONTRACT
}

enum OpWorkApplicationStatus {
  NEW → REVIEWING → INTERVIEW → OFFER → 
  HIRED → REJECTED → WITHDRAWN
}

Модель покрывает: пользователей, профили соискателей и работодателей, резюме с образованием и опытом, вакансии с навыками и тегами, заявки с историей статусов и примечаниями, уведомления.

Кастомный генератор DTO

Одна из интересных технических деталей — prisma-generator-nestjs-dto, кастомный генератор, который автоматически создаёт:

  • DTO для Create/Update операций с class-validator декораторами
  • REST-контроллеры с полной типизацией
  • Data providers для работы с Prisma
  • Формы и списки для фронтенда
  • OpenAPI TypeScript клиент

Это значительно ускоряет разработку: изменил схему Prisma — получил готовый набор DTO, контроллеров и типизированный клиент для фронтенда.

Инфраструктура и деплой

Проект полностью контейнеризирован:

  • docker-compose для PostgreSQL и всех сервисов
  • PM2 для управления процессами backend и frontend
  • Husky pre-commit хуки для форматирования кода
  • Автоматическая генерация OpenAPI-спефикации

Режимы разработки и production максимально упрощены: npm run start:dev поднимает всю инфраструктуру одной командой.

Использование ИИ при разработке

Как и в других моих проектах, значительная часть кода написана с помощью ИИ-ассистентов. Но ключевые архитектурные решения, модель данных и бизнес-логика — полностью мои.

Особенно полезно ИИ помогал с генерацией повторяющегося кода: DTO, валидации, CRUD-операции — всё то, где ручной труд наиболее рутинный.

Текущий статус и планы

Сейчас проект поставлен на паузу — нет времени на активную разработку. Основной функционал уже работает:

  • Авторизация и регистрация с двойным профилем
  • Управление резюме и вакансиями
  • Система откликов и заявок
  • Фильтрация и поиск
  • Уведомления

Изначально планировалось на базе этой базы сделать специализированную платформу для поиска работы и специалистов для NestJS-разработчиков. В планах также были:

  • Переписка между соискателем и компанией
  • Приглашения от компании
  • Дашборды с аналитикой для обеих ролей
  • Production-деплой
